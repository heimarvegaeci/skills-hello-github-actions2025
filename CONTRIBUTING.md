# Contributing to GitHub Actions Skills

Thank you for your interest in contributing! This document provides guidelines for contributing to this learning repository.

## 🎯 Goals

This repository aims to provide clear, practical examples of GitHub Actions workflows while maintaining high code quality and best practices.

## 🚀 Getting Started

1. Fork the repository
2. Create a feature branch (`git checkout -b feature/amazing-workflow`)
3. Make your changes
4. Test your workflows
5. Commit your changes (`git commit -am 'Add amazing workflow'`)
6. Push to the branch (`git push origin feature/amazing-workflow`)
7. Open a Pull Request

## 📝 Guidelines

### Workflow Guidelines
- Keep workflows simple and well-documented
- Use meaningful names for jobs and steps
- Include comments for complex logic
- Test workflows before submitting

### Code Quality
- Follow existing code style and patterns
- Ensure all checks pass in CI
- Include appropriate documentation
- Write clear commit messages

### Documentation
- Update README.md for significant changes
- Document any new workflows or features
- Include examples where helpful
- Keep documentation up-to-date

## 🔍 Code Review Process

1. All submissions require review
2. Maintainers will review your PR
3. Address any feedback promptly
4. Once approved, changes will be merged

## 📞 Questions?

Feel free to open an issue for:
- Questions about GitHub Actions
- Suggestions for improvements
- Bug reports
- Feature requests

## 🙏 Recognition

Contributors will be recognized in the project documentation. Thank you for helping others learn GitHub Actions!
